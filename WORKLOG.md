## 今日研究笔记（2026-04-13 课题2 Day3）

### 气喷剔除物理分析

**豆子物理参数：**
- 质量：0.15g，直径：8mm
- 终端速度：8.28 m/s（空气中自由下落）
- 通道内径：20mm

**T1→T2下落时间（不同间距）：**
| T1-T2距离 | 下落时间 | T2时速度 |
|-----------|----------|----------|
| 20mm | 64ms | 6.8m/s |
| 30mm | 78ms | 7.4m/s |
| 40mm | 90ms | 7.8m/s |
| 50mm | 101ms | 8.1m/s |

**气喷参数（φ2mm喷嘴）：**
| 供气压力 | 喷嘴速度 | 侧向加速度 |
|----------|----------|------------|
| 50kPa | 215m/s | 3769m/s² |
| 100kPa | 304m/s | 7539m/s² |
| 150kPa | 372m/s | 11309m/s² |

→ **最小有效气喷压力：≈0.04kPa**（只需3mm偏移让豆子碰壁）  
→ **50kPa（0.5bar）最低可用，实际建议100-150kPa**

**推荐通道几何尺寸（单文件方案）：**
```
T1 ───── sensor
T2 ───── sensor (trigger bottom cam)
         ↓ 30mm
    [AIR JET nozzle]
         ↓ 29mm
      通道出口
```
- T1-T2间距：40mm
- T2到气喷：30mm
- 气喷到出口：29mm
- 通道总高：~70mm

**时序参数：**
- 电磁阀响应延迟：15ms
- 气喷持续时间：80ms
- 总窗口：95ms
- 豆子到达气喷位置：78ms（T2触发后）
- 豆子离开通道：110ms（T2触发后）
- **时序余量：15ms ✅ 充足**

### 关键发现：吞吐量瓶颈

| 指标 | 数值 |
|------|------|
| 单通道理论处理量 | 2.39 kg/h |
| 振动给料器实际速率 | 0.27 kg/h（30 beans/min） |
| 目标处理量 | ≥2 kg/h |
| 缺口 | 需要7.4×并联通道 |

**结论：** 单文件通道物理上可满足2kg/h，但振动给料器是实际瓶颈。
解决方案：
- A：2-3个并联单文件通道（结构复杂，成本高）
- B：**宽通道（40×20mm）+ 随机分布 + 目标追踪**（推荐，见下方）

### 备选方案：宽通道 + 多目标追踪

```
通道尺寸：40mm宽 × 20mm高 × 150mm长
         ┌──────────────────┐
  入料 → │  ●  ●            │ → 出料
         │     ●   ●        │
         └──────────────────┘
              ↑ 宽通道
```

- 每帧可能有2-4颗豆子
- 用OpenCV连通域分析分别定位每颗
- top/bottom图像对通过光电传感器触发时刻对齐
- 重叠豆子→整帧标记不可靠，由后续节点处理

## 双摄像头方案设计记录（2026-04-13）

### 核心设计约束
- **单文件通道**：强制豆子一颗一颗通过，top/bottom图像对一一对应
- **光电触发**：`T1 → top_capture`，`T2 → bottom_capture`，不依赖时间同步
- **独立阈值**：上下相机独立标定 L\*a\*b\* 阈值
- **气流喷射**：电磁阀控制 0.08秒短气喷，缺陷豆横向吹入废料槽

### 关键设计参数（Day3更新）
- T1-T2间距：40mm → 下落时间90ms
- T2到气喷：30mm → 到达时间78ms
- 气喷持续：80ms，总窗口95ms
- 所需最低压力：50kPa（任何供压均满足）

### ⚠️ 已知问题：吞吐量瓶颈
振动给料器30 beans/min = 0.27 kg/h，远低于2kg/h目标。
**建议升级方案**：宽通道（40×20mm）+ 随机分布 + 多目标追踪算法，可支持更高吞吐量。
