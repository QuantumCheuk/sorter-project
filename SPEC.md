# 生豆分选机 / Green Coffee Bean Sorter
> 项目代号：HUSKY-SORTER-001  
> 版本：v0.7 | 2026-04-28
> 目标：全指标分选（大小/颜色/重量/密度/含水率）+ 分类标签 + 数据输出 + 分批喂入烘豆机

---

## 1. 设计背景

HUSKY-SORTER-001 是数字化干燥链（DDC）的最上游节点，负责：

1. **全维度分选**：大小、颜色、重量、密度、含水率
2. **分类标注**：产区、豆种、处理法、批次等元数据
3. **数据输出**：每批次完整测量数据 → 烘豆机 `/batch/start` 接口
4. **分批喂入**：按设定重量分批输出，对接烘豆机进豆阀

**对标产能：** ≥ HUSKY-ROASTER-001 的处理量（250g/批次，实际运行~8-10批次/小时）  
→ 目标处理量：**≥ 2kg/小时**

---

## 2. 分选指标与技术方案

### 2.1 尺寸分选（Size Sorting）

| 项目 | 说明 |
|------|------|
| 原理 | 振动给料 + 阶梯形孔板（3D打印PLA/PETG） |
| 规格 | 5级目数：16目/15目/14目/13目/12目（对应约1.18-1.70mm） |
| 实现 | 步进电机驱动振动 bowl feeder，旋转到对应孔位 |
| 材质 | 3D打印（PLA+最高精度，层厚0.12mm）或激光切割PMMA |

### 2.2 颜色分选（Color Sorting）— 双摄像头方案

> **设计决策（2026-04-13）：** 采用上+下双摄像头解决单粒豆子正反面检测问题。单文件通道确保 top/bottom 相机拍摄为同一粒。光电商全集触发而非时间同步。

#### 2.2.1 机械结构

```
         [入料斗 + 振动给料]
                ↓
        ┌─────────────────────┐
        │   单文件通道管        │  ← 垂直 3D 打印通道，内径 ~20mm
        │                     │
        │  [T1] ←顶部光电传感器 │  ← 红外光电传感器（触发 top 拍摄）
        │                     │
        │       ● 豆子         │  ← TOP CAM 俯视拍摄
        │                     │
        │  [T2] ←底部光电传感器 │  ← 红外光电传感器（触发 bottom 拍摄）
        │                     │
        │   [底部透明窗口]      │  ← 磨砂亚克力，LED 光穿透
        │                     │
        └─────────────────────┘
              ↓ 落出
         [剔除区 / 合格通道]
```

**关键设计点：**
- **单文件通道**：豆子一颗一颗通过，top/bottom 图像对一一对应
- **光电触发**：`T1_triggered → top_cam.capture()`，`T2_triggered → bottom_cam.capture()`，不依赖时间同步
- **独立光路**：顶部和底部各有独立 LED 光源，分别标定各自 L\*a\*b\* 阈值

#### 2.2.2 硬件配置

| 相机 | 型号 | 用途 | 标定 |
|------|------|------|------|
| Top Camera | HQ Camera IMX477 + M12 6mm 镜头 | 俯视正面 | 独立阈值组 A |
| Bottom Camera | USB Camera（如 C270） | 仰视背面 | 独立阈值组 B |

| 传感器 | 型号 | 数量 | 作用 |
|--------|------|------|------|
| 光电传感器 | 红外光电开关（遮断式） | 2 | T1/T2 触发拍摄 |
| 顶部 LED | 5V USB 环形灯 × 4 | 1组 | 顶部漫反射光源 |
| 底部 LED | 5V USB 环形灯 × 4 | 1组 | 底部漫反射光源（穿透窗口） |

**底部透明窗口：**
- 材质：乳白色磨砂亚克力（透光率 ≥70%，漫反射好）
- 安装方向：磨砂面朝上（避免粉尘直接落在表面）
- 密封：边缘黑色密封胶 + 遮光棉

```python
bean_buffer = {}  # id -> {top_img, bottom_img, top_defect, bottom_defect}

        "bottom": None

    bot_result = detector.detect(bean_buffer[bean_id]["bottom"], mode="bottom")

    # 任意一面有缺陷 → 剔除

        trigger_reject(bean_id)
    else:
        route_to_grade(bean_id, top_result.quality_score)

    del bean_buffer[bean_id]
```

#### 2.2.5 剔除机构（Rejection Mechanism）

**位置：** 单文件通道出口 → 剔除区

**实现方式：气流喷射式（Air Jet Deflector）**

```
  通道出口
      ↓
  [偏转槽] ← 正常豆直线落下
      ↓
  [侧向气喷嘴] ← 缺陷豆被气流横向吹入废料槽
```

| 组件 | 规格 | 数量 |
|------|------|------|
| 微型电磁阀 | 12V 2-way NC（常闭） | 1 |
| 喷嘴 | 3D 打印 PLA，φ2mm 孔径 | 1 |
| 废料仓 | 3D 打印，容量约 100g | 1 |

**工作流程：**
1. Top + Bottom 双相机均判定为"有缺陷" → 触发电磁阀
2. 电磁阀打开 → 0.1-0.2 秒短气喷
3. 气流将豆子横向吹入废料槽
4. 电磁阀关闭 → 正常豆继续直线落下进入合格通道

**Timing 控制（2026-04-13 物理仿真校准）：**
```python
# 豆子物理参数：质量0.15g，直径8mm，终端速度8.28 m/s
# T1→T2：40mm → 90ms；T2→气喷：30mm → 到达78ms（触发后）
# 气喷参数（φ2mm喷嘴，≥50kPa）：
#   只需3mm偏移即可让豆子碰壁，任何供压均满足
#   推荐供气：100-150kPa (1.0-1.5 bar)
air_blast_delay_ms = 15     # 电磁阀响应延迟
air_blast_duration_ms = 80  # 短气喷持续（豆子碰壁后靠重力滑落）
air_blast_total_ms = air_blast_delay_ms + air_blast_duration_ms  # = 95ms
# 豆子离开通道：T2触发后110ms → 时序余量15ms ✅

def on_bottom_sensor(bean_id):
    # ... 判定逻辑 ...
    if is_defective:
        # 气喷在T2触发后15ms打开（豆子到达气喷位置），80ms后关闭
        # 豆子在T2触发后110ms离开通道 → 时序充足
        schedule_air_blast(delay_ms=air_blast_delay_ms, duration_ms=air_blast_duration_ms)
```

> ⚠️ **已知瓶颈**：振动给料器实际速率约30 beans/min = 0.27 kg/h，
> 低于2kg/h目标。解决方案：2-3个并联单文件通道，或采用**宽通道+多目标追踪**方案。

**正常豆（非缺陷）直接落入合格通道，不触发气喷。**

### 2.3 重量分选（Weight Sorting）

#### 2.3.1 设计方案：Buffer Cup串行称重

> **设计决策（2026-04-13）：** 采用Buffer Cup方案，豆子从颜色传感器落下进入称重杯，测量单粒重量后释放。

**工作流程：**
```
颜色传感器出豆 → 进入称重杯 → 50ms稳定 → HX711读数 → 30ms释放 → 下一粒
```

**时序参数：**
| 阶段 | 时间 |
|------|------|
| 豆子下落+冲击 | ~10ms |
| 重量稳定等待 | 50ms |
| HX711采样（5次平均） | 15ms |
| 杯体复位释放 | 30ms |
| **总周期** | **~80ms/粒** |

**性能分析：**
| 指标 | 数值 | 说明 |
|------|------|------|
| 最大可行速率 | 750 beans/min | 单杯方案上限 |
| 2kg/h目标利用率 | 24% | 有充足余量 |
| 检测准确率 | 100% | 仿真结果 |
| 称重误差 | ±1.3mg | 平均绝对误差 |
| 检测分辨率 | ±5.1mg | 最大误差 |

**结论：** 称重系统不是瓶颈！单杯串行方案完全满足2kg/h目标。

#### 2.3.3 软件实现

```python
class WeightMeasurement:
    def __init__(self):
        self.load_cell = LoadCell(config=HX711Config(
            data_pin=5, clock_pin=6, gain=128
        ))
        self.load_cell.calibrate(known_weight_g=100.0)  # 标定
        self.load_cell.tare()  # 去皮
    
    def measure_bean(self) -> Optional[float]:
        """测量单粒豆子重量（Blocking，约80ms）"""
        weight = self.load_cell.read(samples=5)  # 5次平均
        if weight and 0.08 < weight < 0.5:  # 合理范围
            return weight
        return None
    
    def get_statistics(self) -> dict:
        """获取当前批次统计：均值、标准差、分布"""
        return self.load_cell.get_statistics()
```

#### 2.5.3 测量电路

| 方案 | 芯片 | 精度 | 成本 | 推荐 |
|------|------|------|------|------|
| **I2C容值计** | **AD7746 (24-bit)** | **±0.1%** | **¥60** | **✅ 首选** |
| 555振荡器 | NE555 + GPIO计数 | ±0.5% | ¥5 | ⚠️ 备选 |
| 分立运放 | OPA + 比较器 | ±1% | ¥15 | ❌ 精度不足 |

**AD7746规格：**
- I2C 地址：0x48（ADDR=GND）
- 电容分辨率：1 fF RMS
- 输入范围：±4 pF（差分）或 0-8 pF（单端）
- 采样率：10Hz / 50Hz

**⚠️ 重要设计约束（2026-04-25 修正）：**
- AD7746 输入范围仅 ±4pF（差分），任何探头连接电缆的寄生电容都会直接叠加
- RG174 @ 30cm → 30pF ❌；CAT5e @ 30cm → 15pF ❌
- **解决方案：AD7746 模块必须紧贴探头安装（<5cm 引线），Pi 通过 I2C 缓冲器（PCA9600）延长至 1-2 米，或采用插针式模块直接插在探头 PCB 上**

#### 2.5.5 标定方法（烘干法）

**标准参考法（ISO 6673）：**
```
M_wb% = (W_wet - W_dry) / W_wet × 100

步骤：
1. 取约 5g 样品，精确称重 W_wet
2. 105°C 烘干 16h（或 130°C 烘干 2h）
3. 干燥器冷却30min
4. 称重 W_dry
5. 计算 M_wb%
```

**两点线性标定：**
```
C = a + b × M
已知 (M1=5%, C1) 和 (M2=15%, C2) 求解 a, b
```

### 2.6 缓冲料仓 + 螺旋给料（分批控制）

> **设计决策（2026-04-26）：** 8格缓冲仓（按品质A/B/C分级）+ 旋转分配器 + 螺旋给料到烘豆机

#### 2.6.1 机械结构

```
  [缓冲仓入口漏斗]
         ↓
  ┌──────────────────────┐
  │  8格缓冲仓 A1 A2 A3   │  ← 旋转分配器选择落豆格
  │              B1 B2   │
  │              C1 C2 BF│

         ↓ 各格出豆口
  [旋转圆盘分配阀]  ← 步进电机驱动，选择出豆格
         ↓
  [螺旋给料器]     ← φ20mm管，15mm螺距，步进电机

```

| 每转质量 | 1.791 g/rev | 计算值：π×10²×15/1000×0.40×0.95 |
| 驱动电机 | 28BYJ-48 + DRV8833 | microstep=8 |
| 目标转速 | 120 RPM | 214.9 g/min ≈ 12.9 kg/h |
| 250g批次时间 | ~70秒 | 修正值（原错误估算17秒） |
| 步进分辨率 | 512 steps/rev | 64 steps/rev × 8 microstep |

**⚠️ 重要修正（2026-04-26）：** 原设计φ12mm管道+10mm螺距，250g批次仅需17秒（错误）。
修正后φ20mm管道+15mm螺距，250g批次正确时间为~70秒。

#### 2.6.3 控制模块

**文件：** `sorter/motor/spiral_feeder.py`

| 类 | 功能 |
|----|------|
| `SpiralFeeder` | 螺旋给料器（步进控制+可选HX711闭环重量反馈） |
| `RotaryDistributor` | 8位旋转分配阀（分度控制，<200ms切换） |
| `BufferBinController` | 缓冲仓总控制器（状态机+批次分配逻辑） |

**状态机：**
```
IDLE → FILLING → BATCH_READY → DISPENSING → DISPENSE_DONE → COOLDOWN
```

**批次控制流程：**
1. 各站分选的豆按grade进入对应格（A1-3/B1-2/C1-2）
2. 格满时液位传感器报警（推荐电容液位计，¥15/个）
3. 收到烘豆机 `UPSTREAM_READY=1` 信号
4. 分配器旋转到目标格（<200ms）
5. 螺旋给料器以120RPM出豆（~70秒/250g）
6. 每批次完成发送 `BATCH_FEED_COMPLETE` MQTT消息

#### 2.6.4 与烘豆机接口

| 信号 | 方向 | 说明 |
|------|------|------|
| SORTER_READY | DO | 分选机就绪 |
| UPSTREAM_READY | DI | 烘豆机请求接豆（冷却完成） |
| BATCH_START | MQTT PUB | 批次数据JSON → roaster/{id}/batch/input |
| BATCH_FEED_COMPLETE | MQTT PUB | 出豆完成 → sorter/{id}/batch/feed |
| ROASTER_RECEIVING | DI | 烘豆机确认接收 |

**MQTT主题：**
- `sorter/{id}/batch/output` — 批次就绪（BATCH_READY）
- `sorter/{id}/batch/feed` — 出豆控制
- `roaster/{id}/batch/input` — 接收烘豆机信号
- `roaster/{id}/status` — 监听 UPSTREAM_READY

#### 2.6.5 液位检测方案

| 方案 | 成本 | 精度 | 推荐 |
|------|------|------|------|
| 电容液位计 | ¥15/个 | ±5g | ✅ **推荐** |
| 红外对射 | ¥5/个 | ±5mm | 备选 |
| 复用HX711称重 | ¥30/个 | ±0.5g | 精度最高但复杂 |

**推荐：** 电容液位计 × 8（贴仓壁安装，非接触）

### 3.3 批次处理模式（重要设计约束）

**每次只处理单一产地、单一品种、单一处理法的生豆。**

处理流程：
```
1. 操作员在界面设置批次元数据：
   - 产地（国家+产区/农场）
   - 品种
   - 处理法
   - 批次码 / 供应商
   - 品质等级阈值（A/B/C，可配置）

2. 机器执行全指标检测（不区分品质，只测量+记录）

3. 检测完成后：
   - 按品质等级阈值决定哪些进入合格仓、哪些剔除
   - 尺寸/密度/颜色等原始数据全部记录，供后续分析
   - 缺陷豆统一进入废料仓

4. 准备出豆：
   - 操作员设置每批喂入重量（如 250g）和批次数
   - 分批输出至烘豆机
```

**注意：** `quality_class` 是操作员预设的分类阈值（如：缺陷率<1%=A级，<3%=B级），不是机器自动判断的分级。

### 8.1 吞吐量瓶颈与升级路径（v0.7 更新，2026-04-28）

> ⚠️ **重要发现（v0.7, 2026-04-28）：** 当前单通道设计**无法**达到 2kg/h 目标。

**瓶颈分析（`simulation/throughput_bottleneck_analysis.py`）：**

| 阶段 | 当前能力 | 瓶颈级别 |
|------|---------|---------|
| 振动给料 | 30 bpm = 0.27 kg/h | ⚠️ **CRITICAL**（需133 bpm） |
| 颜色检测 | ~200 bpm | ✅ OK |
| 称重系统 | ~300 bpm | ✅ OK |
| 密度分选 | ~200 bpm（5015风扇） | 🟡 MODERATE |
| 含水率检测 | ~300 bpm | ✅ OK |

**多通道扩展模型（并行单文件通道）：**

| 配置 | 吞吐量 | 状态 |
|------|--------|------|
| 1通道 × 30 bpm | 0.27 kg/h | ❌ |
| 2通道 × 30 bpm | 0.54 kg/h | ❌（仍差73%） |
| 3通道 × 50 bpm | **2.70 kg/h** | ✅ **目标达成（35%余量）** |
| 4通道 × 50 bpm | 3.60 kg/h | ✅ 超目标 |

**升级推荐路径（实现 2kg/h）：**

| 阶段 | 动作 | 估算费用 |
|------|------|---------|
| Phase 1 | 增加第2并行通道（¥150） | ¥150 |
| Phase 2 | 3通道全升级 Nema17 @ 50bpm + turbo blower | ¥520 |
| **合计** | **3通道 × 50bpm = 2.70 kg/h ✅** | **¥520** |

**总成本估算：** ¥1,500（原始）+ ¥520（升级）= **¥2,020**（超原预算42%）

**替代方案（宽通道）：** 60mm宽通道 + 多目标追踪 ≈ 1.08 kg/h ❌（下游仍需升级，不推荐）

**结论：** 推荐**3通道并行方案**（更低软件风险，组件成熟）。当前 v0.6 设计为单通道验证机，量产需升级。
